Codec Architecture

The Higgs Codec Tokenizer compresses audio hierarchically — a semantic path for meaning and an acoustic path for texture, quantized together.

Trained with both a semantic loss (against a frozen teacher) and an acoustic loss — tokens carry meaning and sound.

The Autoregressive TTS Model

A ~4B decoder consumes interleaved text and audio tokens — reference text, reference audio, then multi-turn generation grounded on everything before it.

ComponentSpec
Backbone~4B AR decoder · 36 L · hidden 2560 · GQA 32/8
Codebook embedding / headFused single-tensor, tied with text embedding
Context length8,192 tokens (training sequence length)
Audio tokens8 codebooks × 1026 vocab · delay pattern
Sample rate24 kHz
Frame rate25 fps · 40 ms per frame
✕ FLATTEN — read all 8 codebooks per frame: sequence grows 8×
✓ DELAY PATTERN — one step per frame, codebooks offset diagonally

Codebook k+1 trails codebook k by one step — every step still emits 8 tokens in parallel, but the sequence stays T + 7 instead of T × 8. That's the acceleration.

Granular Control via Inline Tags

Adjust emotion, style, and prosody mid-sentence — the tags live inside the text and the model transitions seamlessly between them.
